class users.models.User(*args, **kwargs)¶

Базовые классы: AbstractUser

Модель для заполнения формы регистрации.

Параметры:

  • id (BigAutoField) – Primary key: ID

  • password (CharField) – Пароль

  • last_login (DateTimeField) – Последний вход

  • is_superuser (BooleanField) – Статус суперпользователя. Указывает, что пользователь имеет все права без явного их назначения.

  • username (CharField) – Имя пользователя. Обязательное поле. Не более 150 символов. Только буквы, цифры и символы @/./+/-/_.

  • first_name (CharField) – Имя

  • last_name (CharField) – Фамилия

  • email (EmailField) – Адрес электронной почты

  • is_staff (BooleanField) – Статус персонала. Отметьте, если пользователь может входить в административную часть сайта.

  • is_active (BooleanField) – Активный. Отметьте, если пользователь должен считаться активным. Уберите эту отметку вместо удаления учётной записи.

  • date_joined (DateTimeField) – Дата регистрации

  • role (CharField) – Role

Relationship fields:

Параметры:

  • groups (ManyToManyField to Group) – Группы. Группы, к которым принадлежит данный пользователь. Пользователь получит все права, указанные в каждой из его/её групп. (related name: user_set)

  • user_permissions (ManyToManyField to Permission) – Права пользователя. Индивидуальные права данного пользователя. (related name: user_set)

Reverse relationships:

Параметры:

logentry (Reverse ForeignKey from LogEntry) – All записи в журнале of this пользователь (related name of user)

users.views.home(request)¶

Данная функция возвращает страницу users/home.html

users.views.profile_views(request)¶

Если пользователь зарегистрирован, данная функция возвращает:

  1. страницу users/manager.html - если пользователь зарегистрирован как менеджер;

  2. страницу users/driver.html - если пользователь зарегистрирован как водитель;

  3. страницу users/profile.html - если пользователь не зарегистрирован ни как менеджер, ни как водитель.

Содержание модуля:¶

Директория users отвечает за регистрацию, авторизацию, заполнение профилей сотрудников, отображение главной страницы сайта. В модуле users/urls.py происходит переадресация к приложению main.
